# 12/11 Summary

## Key Decisions
- Switched betting data sources from Covers + Odds-API to SportsBettingDime (SBD) across odds, splits, trends, futures, and props.
- Kept existing tools/interfaces but rewired them to SBD feeds.

## Implementation Highlights
- Odds provider now uses SBD odds feeds for events/odds and SBD gameprops list for props.
- Betting splits now sourced from SBD `bettingSplits` payload (bets% + handle%) for spread/total/ML.
- ATS + O/U trends now sourced from SBD trends endpoint with situational splits (home/away/fav/dog and totals splits).
- Futures now sourced from SBD futures endpoints; chat tool accepts optional `market`/`books`.
- Team name aliasing for matchup detection in splits formatting covers common NBA aliases.
- Chat prompt/source references updated to SBD and config logging now checks `SBD_BOOK_IDS`/`ODDS_BOOKMAKERS`.

## SBD Endpoints Used
- Odds: `https://www.sportsbettingdime.com/wp-json/adpt/v1/{league}-odds`
- Futures markets: `https://www.sportsbettingdime.com/wp-json/adpt/v1/futures/{league}/markets`
- Futures market details: `https://www.sportsbettingdime.com/wp-json/adpt/v1/futures/{league}/markets/{id}`
- Trends: `https://srfeeds.sportsbettingdime.com/v2/trends/{league}`
- Game props list: `https://cdn-sde.sbdfuel.com/gameprops/{league}/list`
